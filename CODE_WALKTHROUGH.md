# MagicMoon コード読解ガイド

このドキュメントは、MagicMoon コーポレートサイトの各ファイルが「何を担当しているか」「どの順番で読むと理解しやすいか」を初学者向けに整理したものです。HTML / CSS / JavaScript は役割が分かれているため、最初に全体像を押さえると、個別のコードがぐっと読みやすくなります。

---

## 1. サイト全体の仕組み

MagicMoon は PHP やデータベースを使わない静的サイトです。ブラウザが HTML、CSS、JavaScript、画像を読み込み、画面を表示します。

```text
ブラウザ
  ↓ index.html / case.html を読む
HTML
  ↓ css/style.css を読み込む
CSS が見た目を整える
  ↓ js/main.js を読み込む
JavaScript がメニュー・スクロール・フォーム演出を動かす
  ↓ image/ の画像を表示する
```

HTML は「文章や画像の構造」、CSS は「見た目」、JavaScript は「動き」を担当します。この3つを分けて読むのが、Web制作コードを理解する近道です。

---

## 2. 最初に読むおすすめ順

1. `README.md`
   - サイト概要、技術構成、ローカル確認方法を確認します。
2. `index.html`
   - トップページ全体の HTML 構造を読みます。`header`、`nav`、`section`、`footer` の役割を押さえます。
3. `case.html`
   - 納入事例ページを読みます。Lightbox2 の読み込みと画像ギャラリーの構造を確認します。
4. `css/reset.css`
   - ブラウザの初期スタイルをそろえる考え方を確認します。
5. `css/style.css`
   - 色、余白、レイアウト、レスポンシブ、アニメーションを確認します。
6. `js/main.js`
   - ハンバーガーメニュー、スクロール演出、フォームバリデーションなどの動きを確認します。
7. `favicon.svg` / `image/`
   - タブアイコンやサイト内画像が、HTML からどのように参照されているか確認します。

---

## 3. ファイル別の説明

### `README.md`

リポジトリ全体の説明書です。サイト概要、使用技術、主な機能、ローカルでの確認方法、制作背景をまとめています。

初学者が見るポイント:
- 何を作ったサイトか
- どのファイルがどの役割を持つか
- 静的サイトなので DB やサーバー処理が不要なこと
- Lightbox2 など外部ライブラリを使っている箇所

---

### `index.html`

トップページを作る HTML ファイルです。利用者が最初に見るページで、会社の雰囲気、事業内容、納入事例、会社情報、問い合わせフォームを表示します。

主な構成:
- `<head>`: 文字コード、タイトル、CSS、Google Fonts、Font Awesome の読み込み
- `<header>`: ロゴとナビゲーション
- `.hero`: メインビジュアルとキャッチコピー
- `.news`: お知らせ一覧
- `.business`: 事業内容のカード
- `.case`: 納入事例への導線
- `.company`: 会社情報
- `.contact`: 問い合わせフォーム
- `<footer>`: コピーライトや下部情報

初学者が見るポイント:
- `id="business"` などはページ内リンクの移動先になる
- `class="..."` は CSS や JavaScript から要素を指定するための名前
- `header`、`nav`、`section`、`footer` を使うと、ページ構造の意味が分かりやすい
- 問い合わせフォームの `id="contactForm"` は `js/main.js` から使われる

---

### `case.html`

納入事例ページです。トップページの「more」リンクなどから移動し、具体的な事例と画像ギャラリーを表示します。

主な構成:
- `header`: トップページと共通のナビゲーション
- `.page-header`: ページタイトル
- `.case-detail`: 納入事例の詳細説明
- `.gallery`: 画像ギャラリー
- `data-lightbox="gallery"`: Lightbox2 で画像をグループ化する設定
- ページ末尾の `<script>`: jQuery、Lightbox2、`js/main.js` の読み込み

初学者が見るポイント:
- 画像をクリックして拡大する動きは、自作ではなく Lightbox2 ライブラリが担当している
- `data-lightbox` や `data-title` は、ライブラリへ情報を渡すための属性
- `case.html` も `css/style.css` と `js/main.js` を共有している

---

### `css/reset.css`

ブラウザごとの差を減らすための CSS です。ブラウザは何も指定しなくても見出しやリストに初期スタイルを持っています。その差を一度そろえる目的で使います。

主な処理:
- `margin`、`padding`、`border` をリセットする
- `box-sizing: border-box` でサイズ計算を分かりやすくする
- 画像やテーブルなどの基本表示を整える

初学者が見るポイント:
- リセット CSS は「デザインする前の土台作り」
- `style.css` より先に読み込むと、後から自分のスタイルを上書きしやすい
- このプロジェクトでは `style.css` 側にも一部リセットがあるため、補助的な役割

---

### `css/style.css`

サイト全体の見た目を担当するメイン CSS です。色、余白、配置、アニメーション、レスポンシブ対応など、見た目に関するほとんどの処理がここにあります。

主な構成:
- `:root`: ゴールド、黒、文字色、影、余白などの CSS 変数
- 共通設定: `body`、リンク、画像、コンテナ
- ヘッダー: ロゴ、ナビゲーション、ハンバーガーメニュー
- トップページ: ヒーロー、お知らせ、事業内容、納入事例、会社情報、問い合わせ
- 事例ページ: 詳細セクション、ギャラリー、Lightbox の見た目調整
- アニメーション: フェードイン、ホバー、スクロール演出
- `@media`: 画面幅ごとのレスポンシブ対応

初学者が見るポイント:
- `var(--primary-gold)` のように CSS 変数を使うと、色を一括管理できる
- Flexbox は横並びやナビゲーション、Grid はカードやギャラリー配置に向いている
- JavaScript が付ける `.active` や `.scrolled` クラスに対して、CSS が見た目を変えている
- `@media` の中は、スマホやタブレット向けの上書き設定

---

### `js/main.js`

サイトに動きを加える JavaScript ファイルです。HTML と CSS だけではできない「クリックに反応する」「スクロールに反応する」「フォーム入力をチェックする」といった処理を担当します。

主な処理:
- `DOMContentLoaded`: HTML 読み込み後に JS を実行する準備
- ハンバーガーメニュー: モバイル用ナビゲーションの開閉
- ヘッダースクロール効果: スクロール量に応じて `.scrolled` クラスを付ける
- スムーススクロール: ページ内リンクを滑らかに移動する
- `IntersectionObserver`: 要素が画面に入ったタイミングでフェードインする
- 問い合わせフォーム: 入力チェックと通知表示
- パララックス効果: ヒーロー画像をスクロール量に合わせて動かす

初学者が見るポイント:
- JavaScript は `id` や `class` を使って HTML 要素を探す
- `addEventListener()` は「何かが起きたら処理する」を表す基本形
- `classList.add()` / `remove()` / `toggle()` で CSS の状態クラスを切り替える
- 通知ポップアップは JavaScript が HTML 要素を動的に作っている

---

### `favicon.svg`

ブラウザのタブやブックマークに表示される小さなアイコンです。SVG は XML ベースの画像形式で、HTML に近いタグ構造で図形を描けます。

主な構成:
- `<svg>`: SVG 全体のキャンバス
- `<rect>`: 背景の角丸四角形
- `<circle>`: 月や星の形を作る円
- `fill`: 図形の色

初学者が見るポイント:
- SVG は画像ファイルですが、テキストで中身を読める
- `viewBox="0 0 64 64"` は 64 × 64 の座標空間として描く指定
- HTML から `<link rel="icon" href="favicon.svg">` で読み込まれる

---

### `image/`

サイト内で表示する画像素材のフォルダです。HTML の `<img src="image/...">` や CSS の背景画像から参照されます。

主な画像:
- `メインスライド.jpg`: トップページのメインビジュアル
- `メインスライド2.jpg`: 補助的なビジュアル
- `BtoB空間デザインサポート.jpg`: 事業内容カード用
- `パーソナルソーシング.jpg`: 事業内容カード用
- `ヨーロッパ家具.jpg`: 事業内容カード用
- `納入事例ヘッダー.jpg`: 事例ページのヘッダー画像
- `ギャラリー1.jpg` など: Lightbox ギャラリー用

初学者が見るポイント:
- 画像ファイル名と HTML の `src` が一致していないと表示されない
- 日本語ファイル名は環境によって扱いに注意が必要
- 画像はコードではないため、説明は README やこのガイドにまとめておくと親切

## 4. 代表的な処理の追い方

### ハンバーガーメニュー

```text
index.html / case.html
  ↓ id="hamburger" と id="nav"
js/main.js
  ↓ クリック時に active クラスを付け外し
css/style.css
  ↓ .hamburger.active / .nav.active の見た目が適用される
```

### スムーススクロール

```text
ナビリンク <a href="#business">
  ↓ クリック
js/main.js
  ↓ クリック標準動作を止め、移動先の位置を計算
window.scrollTo({ behavior: "smooth" })
  ↓
なめらかに移動
```

### フェードイン演出

```text
HTML の .news-item / .business-item など
  ↓
js/main.js の IntersectionObserver
  ↓ 画面内に入ると visible クラスを追加
css/style.css
  ↓ opacity や transform が変わって表示される
```

### お問い合わせフォーム

```text
index.html の form#contactForm
  ↓ submit
js/main.js
  ↓ 入力漏れとメール形式をチェック
showNotification()
  ↓
成功/エラー通知を画面右上に表示
```

### Lightbox ギャラリー

```text
case.html の <a data-lightbox="gallery">
  ↓ 画像クリック
Lightbox2 ライブラリ
  ↓
画像を拡大表示し、前後の画像へ切り替え可能にする
```

---

## 5. 学習時に意識するとよいこと

- HTML はページの意味構造を作る
- CSS は見た目とレイアウトを作る
- JavaScript はユーザー操作に反応する動きを作る
- `class` は CSS と JavaScript の両方から使われる重要な名前
- `id` はページ内で1つだけの要素を指すときに使う
- 外部ライブラリは CDN の `<link>` や `<script>` で読み込める
- レスポンシブ対応は `@media` の中にある上書き設定を見る
- 画像やフォントも、サイトの印象を作る重要な部品
